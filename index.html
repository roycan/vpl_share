Unlock Efficient & Effective Programming Assessment with VPL

A comprehensive guide for high school Computer Science teachers looking to enhance programming education through Moodle's Virtual Programming Lab.

Key Benefits

  • Automated assessment saves hours of grading time
  • Immediate feedback helps students learn faster
  • Built-in tools to address AI-generated code and plagiarism

Learn About VPL Implementation Guide

What is VPL & Why Use It?

Virtual Programming Lab is a powerful Moodle plugin that transforms how programming is taught and assessed.

Core Functionalities

Interactive Code Editor

Built-in editor with syntax highlighting, auto-completion, and real-time validation.

Automated Execution

Run student code in a secure environment with customizable resource limits.

Auto-Grading

Define test cases to automatically evaluate student solutions and provide immediate feedback.

Plagiarism Detection

Built-in tool to identify similarities between student submissions.

Benefits for Teachers

  • Efficiency in Grading

    Automatically check for common errors like missing semicolons, infinite loops, or incorrect output formats.

  • Consistent Assessment

    Apply the same evaluation criteria to all submissions, ensuring fairness.

  • Detailed Analytics

    Track submission attempts, identify common mistakes, and understand student approaches.

  • Flexible Test Cases

    Create public test cases for guided learning and hidden test cases for thorough evaluation.

Benefits for Students

  • Instantaneous Feedback

    Get immediate results after submission, no waiting for teacher evaluation.

  • Iterative Learning

    Try multiple approaches, learn from mistakes, and resubmit without penalty.

  • Error Identification

    Precise error messages help pinpoint and understand mistakes.

  • Independent Problem-Solving

    Develop debugging skills and build confidence in code analysis.

VPL in the Modern Classroom

Navigating AI and Academic Integrity with VPL

Addressing AI-Generated Code

As AI tools like ChatGPT become more prevalent, VPL provides mechanisms to ensure students truly understand programming concepts rather than simply pasting AI-generated solutions.

  • Create complex, context-specific problems that require true understanding
  • Implement hidden test cases that evaluate edge conditions
  • Require code explanations or documentation in submissions

Combating Plagiarism

VPL's built-in similarity detection helps identify potential plagiarism by comparing code structure and patterns across student submissions.

  • Automatic detection of similar code between students
  • Visual reports highlighting matching code segments
  • Customizable detection thresholds for different assignments

Focus on Skill Development

Beyond just evaluating final solutions, VPL helps students build crucial programming skills applicable in any environment.

  • Debugging skills through specific error messages
  • Critical thinking via test case analysis
  • Self-assessment through immediate feedback

Designing AI-Resistant Assignments

When creating VPL assignments, focus on problems that require multiple approaches, integration of concepts, or explanation of reasoning.

A well-designed VPL assignment tests not just the final output but the student's ability to interpret requirements, structure solutions, and handle edge cases â€“ skills that demonstrate true understanding beyond what AI can generate.

Implementation Guide

Follow these general steps to set up VPL in your Moodle environment

Installing VPL in Moodle

  • Administrator access to your Moodle installation
  • Moodle version 3.9 or higher (recommended)
  • PHP version 7.2 or higher
  1. Download the VPL plugin from Moodle plugins directory.
  2. Log in to Moodle as admin and install the plugin.
  3. Configure VPL settings in Site administration > Plugins > Activity modules > Virtual Programming Lab.
  4. Create your first VPL assignment in a course.

Setting Up VPL Jail Server (Linode)

  • Choose a Linode plan (Nanode 1GB for small classes, 2GB+ for larger)
  • Deploy Ubuntu 20.04 LTS
  • SSH into your server and update: sudo apt update && sudo apt upgrade -y
  • Install dependencies: sudo apt install git build-essential -y
  • Clone and install VPL Jail: git clone https://github.com/jcrodriguez-dis/vpl-jail-system.git then cd vpl-jail-system && sudo ./install-vpl-sh

Securing with Let's Encrypt

  • Register a domain and point DNS to your server
  • Open ports 80 and 443 in your firewall
  • Install Nginx: sudo apt install nginx -y
  • Install Certbot: sudo apt install certbot python3-certbot-nginx -y
  • Run Certbot: sudo certbot --nginx and follow prompts

Java - Object-Oriented Programming

Template for Java assignments focusing on OOP concepts with a class implementation.

Template Files

# Student Files (to be edited by students):
#   Student.java
#   Main.java (can be provided partially or fully by teacher)
#
# Teacher Files (to be provided/configured by teacher):
#   vpl_evaluate.cases

Student.java

// Student.java
public class Student {
    private String name;
    private int id;
    private double gpa;
    
    // TODO: Implement the constructor
    public Student(String name, int id, double gpa) {
        this.name = name;
        this.id = id;
        this.gpa = gpa;
    }
    
    // TODO: Implement getters and setters
    public String getName() {
        return this.name;
    }
    
    public int getId() {
        return this.id;
    }
    
    public double getGpa() {
        return this.gpa;
    }

    public void setName(String name) {
        this.name = name;
    }

    public void setId(int id) {
        this.id = id;
    }

    public void setGpa(double gpa) {
        this.gpa = gpa;
    }
    
    // TODO: Implement a method to check if the student is in good standing
    // A student is in good standing if their GPA is at least 2.0
    public boolean isInGoodStanding() {
        return this.gpa >= 2.0;
    }
}

Main.java (Example)

// Main.java
// Students might fill this in, or teacher provides it for testing.
public class Main {
    public static void main(String[] args) {
        // Test case 1
        Student student1 = new Student("Alice", 101, 3.5);
        System.out.println("Name: " + student1.getName());
        System.out.println("ID: " + student1.getId());
        System.out.println("GPA: " + student1.getGpa());
        System.out.println("Good Standing: " + student1.isInGoodStanding());

        // Test case 2
        Student student2 = new Student("Bob", 102, 1.9);
        System.out.println("Name: " + student2.getName());
        System.out.println("ID: " + student2.getId());
        System.out.println("GPA: " + student2.getGpa());
        System.out.println("Good Standing: " + student2.isInGoodStanding());
    }
}

// validator.js
// Write functions to validate form input

/**
 * Validates an email address format
 * @param {string} email - The email to validate
 * @return {boolean} - True if email is valid, false otherwise
 */
function validateEmail(email) {
    // TODO: Implement email validation
    // A valid email should have a username, @ symbol, and domain
    if (!email) return false;
    const emailRegex = /^[^\s@]+@[^\s@]+\.[^\s@]+$/;
    return emailRegex.test(email);
}

/**
 * Validates a password (example: at least 8 chars)
 */
function validatePassword(password) {
    // TODO: Implement password validation
    return typeof password === 'string' && password.length >= 8;
}

    Frequently Asked Questions

    Common questions about VPL implementation and usage

    VPL supports a wide range of programming languages, including but not limited to:

    • C/C++
    • Java
    • Python
    • JavaScript
    • PHP
    • Ruby
    • C#
    • Fortran
    • SQL
    • VHDL
    • Scala
    • Haskell

    Additional languages can be added by configuring the jail server environment and execution scripts.

    The cost of running a Linode server for VPL jail depends on the plan you choose:

    • Nanode 1GB: Approximately $5/month, suitable for small classes or testing.
    • Linode 2GB: Approximately $10/month, good for medium-sized classes.
    • Linode 4GB: Approximately $20/month, recommended for larger deployments or resource-intensive languages.

    Educational institutions may be eligible for discounted rates or credits from Linode. Check their website for current educational programs.

    The VPL jail server requires minimal maintenance:

    • Regular Updates: Apply system updates monthly for security patches.
    • SSL Certificate Renewal: Let's Encrypt certificates auto-renew, but monitor for any issues.
    • VPL Updates: Occasionally check for VPL jail system updates via Git.
    • Log Monitoring: Periodically check logs for errors or unusual activity.

    In practice, once properly set up, the jail server can run with minimal intervention, requiring perhaps 1-2 hours of maintenance per month.

    • Additional Software: Install libraries, compilers, or tools needed for specific courses.
    • Execution Scripts: Modify the default execution scripts in the VPL configuration to change how programs are compiled and run.
    • Resource Limits: Configure memory limits, execution time, and other constraints globally or per-assignment.
    • Custom Test Frameworks: Integrate with testing frameworks like JUnit, pytest, or custom grading systems.

    The jail server runs in a Linux environment, so you have considerable flexibility to customize it for your specific educational needs.

    If Moodle cannot connect to your VPL jail server, check these common issues:

    1. Firewall Settings: Ensure the jail server port (default 2000) is open in your firewall.
    2. Service Status: Verify the VPL jail service is running with sudo systemctl status vpl-jail-system.
    3. Correct URL: Double-check the URL in Moodle settings (http://yourip:port or https://yourdomain).
    4. Matching Passwords: Ensure the HTTP password in Moodle matches the one in the jail server configuration.
    5. Network Issues: Check if your school network blocks outgoing connections to the jail server.
    6. SSL Certificates: If using HTTPS, verify the certificate is valid and trusted.
    7. Logs: Check VPL jail logs with sudo journalctl -u vpl-jail-system for error messages.

    You can test the connection from the command line with: curl -I http://yourip:port to see if the server is responding.
