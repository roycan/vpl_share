Implementation Guide

A step-by-step approach to setting up VPL in your classroom

VPL Implementation Overview

Implementing VPL consists of several key steps. This guide will walk you through each one, from installing the Moodle plugin to configuring the jail server and creating your first programming assignment.

1

Install VPL Plugin in Moodle

The first step is to install the VPL plugin in your Moodle installation.

  1. Log in to your Moodle site as an administrator
  2. Go to Site administration > Plugins > Install plugins
  3. Download the VPL plugin from Moodle plugins directory
  4. Upload the ZIP file in the plugin installer
  5. Follow the on-screen instructions to complete the installation
  6. Once installed, go to Site administration > Plugins > Activity modules > Virtual Programming Lab to configure the settings
2

Set Up a Jail Server (Optional but Recommended)

The jail server securely executes student code. While not strictly required for basic VPL functionality, it's strongly recommended for security and to enable advanced features.

Basic Jail Server Setup:

  1. Choose a Linux server (physical or virtual) with at least 2GB RAM and 20GB storage
  2. Install required packages: 
    sudo apt-get update
sudo apt-get install apache2 php php-zip php-mbstring php-xml wget unzip
  3. Download and install the jail server: 
    wget https://github.com/jcrodriguez-dis/vpl-jail-system/archive/refs/heads/master.zip
unzip master.zip
cd vpl-jail-system-master
sudo ./install-vpl-jail-system.sh
  4. Configure the jail server by editing its configuration file: 
    sudo nano /etc/vpl/vpl-jail-system.conf
  5. Start the jail server: 
    sudo systemctl start vpl-jail-system

Advanced Configuration (SSL/TLS):

For production environments, you should secure your jail server with SSL/TLS:

  1. Generate self-signed certificates or obtain certificates from a trusted CA
  2. Configure the jail server to use SSL by editing the configuration file
  3. Update the Moodle VPL settings to connect to the jail server using HTTPS

Detailed instructions for SSL/TLS setup can be found in the official jail server documentation.

3

Configure VPL in Moodle

After installation, you need to configure VPL to work with your jail server (if you set one up).

  1. Go to Site administration > Plugins > Activity modules > Virtual Programming Lab
  2. Set the jail servers URL(s) in the format: 
    http://your-jail-server:8080
    Or if using SSL: 
    https://your-jail-server:443
  3. If you have multiple jail servers, list them separated by commas
  4. Configure other settings according to your preferences
  5. Save changes
4

Create Your First VPL Assignment

Now you can create your first programming assignment using VPL.

  1. Go to your course
  2. Turn editing on
  3. Click "Add an activity or resource" and select "Virtual Programming Lab"
  4. Fill in the basic settings (name, description, grade options, etc.)
  5. Configure execution options:
    • Run: Enable to allow students to run their code
    • Debug: Enable to allow debugging
    • Evaluate: Enable for automatic evaluation
    • Automatic evaluation: Set the behavior of automatic evaluation
  6. Set submission restrictions if needed (file types, size, etc.)
  7. Click "Save and display"
5

Set Up Evaluation Scripts

To enable automatic evaluation, you need to create evaluation scripts.

  1. In your VPL assignment, go to "Execution files"
  2. Create the following files:
    • vpl_run.sh: Script to run the program
    • vpl_evaluate.sh: Script for evaluation
    • vpl_test.cases: Test cases for assessment
  3. Example vpl_run.sh for Python: 
    #!/bin/bash
python3 $VPL_SRCFILE
  4. Example vpl_evaluate.sh for basic testing: 
    #!/bin/bash
# Load common script and grade functions
. vpl_environment.sh
#

# Compile the student's submission if needed
# (not necessary for Python)

# Run the program with test cases
cat > vpl_execution << "END_SCRIPT"
#!/bin/bash
python3 $VPL_SRCFILE < vpl_test.cases
END_SCRIPT

# Run the test cases
chmod +x vpl_execution
./vpl_execution > vpl_execution.out

# Check output against expected output
# Your evaluation logic here

# Example grade assignment
VPL_GRADEMAX=100
VPL_GRADE=100

# Display feedback
echo "Your program has been evaluated."
echo "Grade: $VPL_GRADE/$VPL_GRADEMAX"
  5. Example vpl_test.cases: 
    10 20
30 40
-5 5
  6. Save the files

These are basic examples. For more advanced evaluation scripts, check out our VPL Templates section.

6

Test Your VPL Assignment

Before making the assignment available to students, test it thoroughly.

  1. Create a test submission
  2. Try running, debugging, and evaluating the submission
  3. Verify that the evaluation script works as expected
  4. Check that the grading criteria are applied correctly
  5. Make any necessary adjustments to the scripts
7

Introduce VPL to Your Students

Finally, introduce VPL to your students with clear instructions.

  1. Create a tutorial or demonstration for students
  2. Explain how to submit assignments, run code, and interpret feedback
  3. Start with simpler assignments to allow students to get familiar with the system
  4. Provide support channels for technical questions
